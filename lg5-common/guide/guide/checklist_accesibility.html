Mobile 웹접근성 준수사항

기본 사항

- 해당 프로젝트는 웹접근성 인증마크 획득을 목표로 진행됩니다.(PC / Mobile Web / Mobile App)

- 검수 및 수정 요청의 주는 CJ 내부검수팀이며, 제공하는 가이드에 따라 진행합니다.(엄격하게 진행됩니다)

1. alt 값

- 꾸밈 용도로 사용된 이미지의 경우 빈 alt값만 제공한다. Ex) alt=""

2. 레이블 입력

- 입력 서식의 경우 label 또는 title을 사용하여 레이블을 부여한다.

3. 링크 텍스트 입력

- 기능이 있는 링크의 경우 적절한 텍스트 정보를 제공한다.
- 동일한 링크텍스트를 제공하더라도 그 내용이 다르다면 각각 자세한 정보 텍스트를 추가하여 제공한다. Ex) read more

4. 새 창 알림

- 새창의 경우 target="_blank"와 title="새창" 을 함께 기재

5. iOS VoiceOver 체크사항

- 키보드 접근 오류로 인해 font-size:0 속성을 사용하지 않는다.
- 초점 범위가 보다 정확히 잡히도록 a태그에 role="button" 속성을 기재한다.
- 페이지 내 아이디 위치 이동의 경우 해당 아이디를 가지는 dom 내부에 한가지 이상의 태그 존재 시 이동 오류가 발생하므로 위치 이동을 위한 숨김 dom을 생성.
